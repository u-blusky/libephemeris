# LibEphemeris v0.23.0 — LEB Precision V3

**Release date:** 2026-03-09

## Headline

**Sub-milliarcsecond precision for all 31 celestial bodies across all three
LEB tiers.** The LEB binary ephemeris runtime pipeline has been completely
rewritten, achieving a 14,600x improvement in worst-case accuracy.

| Metric | Before (v0.22.0) | After (v0.23.0) |
|--------|-------------------|------------------|
| Worst-case error | 4.85" (Saturn) | 0.000332" (Moon) |
| Bodies passing <0.001" | 9/31 | **31/31** |
| Tiers at <0.001" | 0/3 | **3/3** |

### Bodies NOT in LEB (transparent Skyfield fallback)

These bodies are fully supported by libephemeris but are not stored in LEB
files. When requested, they silently fall back to the Skyfield pipeline
(~120 µs/eval) with identical accuracy:

- **Pholus** (ID 16) — the only gap in the 0–22 range
- **Additional hypotheticals** (IDs 51–58) — Leverrier, Adams, Lowell, Pickering, Vulcan, Selena, Proserpina, Waldemath
- **TNOs** — Eris, Sedna, Haumea, Makemake, Quaoar, Orcus, Ixion, Gonggong, Varuna (~9 bodies)
- **Additional asteroids** — Nessus, Asbolus, Chariklo, Apophis, Hygiea, and ~17 others
- **Fixed stars** — 102 stars (stored in LEB star catalog section for sidereal ayanamsha, but position calculations use Skyfield)
- **Planetary moons** — 21 moons (Io, Europa, Ganymede, Callisto, Titan, Triton, Charon, etc.)
- **Astrological angles and Arabic parts** — Ascendant, MC, Pars Fortunae, etc. (derived from house calculations)

Fallback also triggers for unsupported flags (`SEFLG_TOPOCTR`, `SEFLG_XYZ`,
`SEFLG_RADIANS`, `SEFLG_NONUT`) and JDs outside the LEB file's date range.

### New runtime pipeline (Pipeline A')

The previous LEB pipeline stored geocentric ecliptic coordinates directly in
Chebyshev polynomials. This approach failed for two fundamental reasons:

1. **Retrograde cusps** — planets near retrograde stations have sharp reversals
   in ecliptic longitude that Chebyshev polynomials cannot approximate
2. **COB oscillations** — outer planet "center" positions include high-frequency
   oscillations from their moons, creating fitting noise

The new pipeline stores smooth **ICRS barycentric** positions (or system
barycenter positions for outer planets) and applies the full astrometric
correction chain at runtime:

```
Chebyshev coefficients (ICRS barycentric)
    → COB correction (system barycenters only)
    → Light-time iteration
    → PPN gravitational deflection (Sun/Jupiter/Saturn)
    → Special-relativistic aberration
    → IAU 2006/2000A precession-nutation
    → Geocentric ecliptic of date
```

### Key technical changes

**New coordinate type `COORD_ICRS_BARY_SYSTEM`** — Jupiter through Pluto now
store system barycenter positions. System barycenters are determined purely by
solar system gravitational dynamics, producing ultra-smooth trajectories ideal
for Chebyshev approximation. The planet-center offset (COB) is applied at
runtime via SPK data or analytical moon theories.

**PPN gravitational deflection** — Light from distant objects is deflected by
the Sun (~1.75" max), Jupiter (~0.017"), and Saturn (~0.006"). The previous
pipeline omitted this, causing a 3.95" error on Saturn. The new pipeline
applies the standard post-Newtonian formula for all three deflectors.

**Skyfield-compatible asteroid pipeline** — New `_SpkType21Target` class wraps
SPK Type 21 asteroid kernels as Skyfield `VectorFunction` objects, routing
asteroids through Skyfield's `observe()`/`apparent()` pipeline. This eliminated
a 0.3–0.4" systematic error from frame tie and nutation model differences
between the manual ecliptic J2000 approach and Skyfield's ICRS pipeline.

**Ecliptic body parameter tuning** — OscuApogee, InterpApogee, and
InterpPerigee tightened from 8-day/degree-13 to 4-day/degree-15 intervals,
reducing errors from ~0.028" to ~0.000049".

**Delta T precision fix** — Replaced the LEB reader's linearly-interpolated
sparse Delta T table (up to ~0.004s error) with Skyfield's precise model,
eliminating ~0.002" Moon errors from imprecise UT→TT conversion.

## Bugs fixed

- **COB evaluation time bug** — COB correction was evaluated at retarded time
  instead of observer time, causing residual errors for outer planets
- **Asteroid pipeline mismatch** — Asteroids used manual ecliptic J2000 +
  precession instead of Skyfield's ICRS pipeline (~0.3–0.4" systematic error)
- **Ecliptic body generation bug** — `generate_ecliptic_bodies_vectorized()`
  hardcoded `interval_days=8, degree=13` for all ecliptic bodies, ignoring
  per-body `BODY_PARAMS`
- **`measure_precision.py` mode-switching bug** — Reference `swe_calc()` calls
  could silently use LEB via auto-discovery; `LEBReader` was recreated per
  sample; cross-tier SPK mismatch caused false 0.05" errors

## Breaking changes

None. The LEB binary format version remains at 1. Existing `.leb` files
generated with v0.22.0 will still load, but should be regenerated to benefit
from the precision improvements. The public API is unchanged.

## Upgrade guide

1. Update libephemeris: `pip install libephemeris==0.23.0`
2. Regenerate LEB files (recommended):
   ```bash
   poe leb:generate:base:groups      # ~38 seconds
   poe leb:generate:medium:groups    # ~3 minutes
   poe leb:generate:extended:groups  # ~45 minutes
   ```
   Or download pre-generated files:
   ```bash
   libephemeris download:leb:base
   libephemeris download:leb:medium
   ```
3. No code changes required — the API is identical.
