## What 1.0.0 Means

LibEphemeris 1.0.0 is the first stable release. It marks the point where the
public API surface, precision characteristics, backend architecture, and binary
ephemeris formats are considered production-ready and subject to semantic
versioning guarantees.

The library provides a 1:1 API-compatible replacement for
[pyswisseph](https://github.com/astrorigin/pyswisseph), the Python wrapper
around the Swiss Ephemeris. All `swe_*` functions are implemented with matching
signatures and return types. The underlying computation uses NASA JPL
DE440/DE441 planetary ephemerides via Skyfield and IAU 2006/2000A standards via
pyerfa -- an entirely independent codebase with no Swiss Ephemeris source code.

This release encompasses the full development arc from initial compatibility
work through the 0.20-0.26 series and 15 alpha prereleases (1.0.0a1-a15),
consolidating all features, precision improvements, performance optimizations,
and bug fixes into a single stable baseline.

## Project Recap: From First Compatibility Work To 1.0.0

The project began as an effort to build a pure-Python astronomical ephemeris
library with full Swiss Ephemeris API compatibility, using modern data sources
(NASA JPL DE440/DE441) and modern standards (IAU 2006/2000A) instead of the
Swiss Ephemeris's internal planetary theory.

### Early versions (0.20-0.22): Foundation and LEB

The initial PyPI-published versions established the core `swe_*` API surface
and basic Skyfield-backed calculations. Version 0.22.0 introduced the Binary
Ephemeris Mode (LEB) system -- precomputed Chebyshev polynomial approximations
for 31 celestial bodies stored in `.leb` files, providing ~14x runtime speedup
over the Skyfield/JPL pipeline. This release also added REBOUND/ASSIST N-body
integration for asteroid orbit propagation and the group generation workflow
to avoid macOS multiprocessing deadlocks.

### Version 0.23.0: LEB precision breakthrough

The LEB runtime pipeline was completely rewritten, replacing the previous
ICRS-to-ecliptic coordinate conversion approach (which suffered from 1-5"
errors due to retrograde cusps and center-of-body oscillations) with a
physics-correct pipeline storing smooth ICRS barycentric coordinates and
applying gravitational deflection, stellar aberration, and precession-nutation
at evaluation time. Worst-case error dropped from 4.85" (Saturn) to 0.000332"
(Moon) -- a 14,600x improvement.

### Version 0.24.0: Full API audit

The most thorough correctness audit in the project's history: 42 commits
covering every major subsystem. Hypothetical bodies improved from ~2 deg to
~35" (200x). SEFLG_NONUT modes from 18.9" to <0.6" (30x). Fixed stars J2000
frame from ~5.4" to <0.01" (500x+). Lunar eclipse magnitude errors up to 2.2
were eliminated. New flags (SEFLG_NOGDEFL, SEFLG_ICRS, SEFLG_SPEED3), Saros
series in eclipse attributes, 4 missing sidereal modes, dynamic planet radii
for eclipses, and full fixed-star flag support.

### Versions 0.25-0.26: Sidereal precision and hyper-validation

Four critical sidereal coordinate transform bugs were fixed, 537 new sidereal
regression tests added, and 134,000+ deep validation test cases executed across
160 waves covering every API function, all 31 LEB bodies, all 44 ayanamsha
modes, all house systems, eclipses, occultations, heliacal events, crossing
functions, and numerical edge cases. The hyper-validation against pyswisseph
reached 4,400 comparison rounds: 3,947 PASS, 441 KNOWN (inherent engine
divergences), 0 FAIL, 0 ERROR.

## LEB And LEB2

### LEB (LibEphemeris Binary)

LEB files store precomputed Chebyshev polynomial approximations for 31
celestial bodies in four groups:

| Group | Bodies | Count |
|-------|--------|-------|
| Core | Sun-Pluto, Earth, Mean/True Node, Mean Apogee | 14 |
| Asteroids | Chiron, Ceres, Pallas, Juno, Vesta | 5 |
| Apogee | Osculating Apogee, Interpolated Apogee/Perigee | 3 |
| Uranians | Cupido, Hades, Zeus, Kronos, Apollon, Admetos, Vulkanus, Poseidon, Transpluto | 9 |

Three precision tiers:

| Tier | Date range | LEB1 size | LEB2 size |
|------|-----------|-----------|-----------|
| Base | 1850-2150 CE | 53.1 MB | ~33 MB |
| Medium | 1550-2650 CE | 174.6 MB | ~119 MB |
| Extended | -5000 to +5000 CE | 1603.6 MB | ~897 MB |

LEB provides ~14x speedup over Skyfield. Precision is <0.001" vs Skyfield for
all bodies across all tiers. Bodies not in LEB transparently fall back to
Skyfield with no user-visible difference.

The LEB runtime pipeline stores ICRS barycentric coordinates and applies at
evaluation time: PPN gravitational light deflection (Sun + Jupiter + Saturn),
special-relativistic stellar aberration, and IAU 2006 precession-nutation.
Outer planets store system barycenter positions with center-of-body correction
applied at runtime.

### LEB2 compressed format

LEB2 uses error-bounded lossy compression to achieve 4-10x reduction vs LEB1
while maintaining <0.001" precision. The compression pipeline:

1. **Mantissa truncation** -- zero out unneeded mantissa bits per Chebyshev
   coefficient order. High-order coefficients below the precision target are
   zeroed entirely.
2. **Coefficient-major reorder** -- group same-order coefficients across all
   time segments for better compression locality.
3. **Byte shuffle** -- HDF5/Blosc-style byte-lane transposition so exponent
   bytes and zeroed mantissa bytes cluster together.
4. **zstd level 19** -- the regularized data compresses at high ratio.

Per-body precision targets are adaptive: Moon and Earth use 1e-12 AU (small
geocentric distance amplifies errors through light-time, deflection, and
aberration), inner planets use 1e-10 AU, and other bodies use 5e-9 AU.

The v2 chunked format (introduced in a13) splits each body's coefficients into
10-year temporal chunks compressed independently. On first access only the ~300
KB chunk containing the requested Julian Day is decompressed, reducing cold
start from 1568ms (v1) to 47ms (v2).

**LEB2 base-tier core ships inside the PyPI wheel.** The 14 core bodies
(~10.6 MB, 1850-2150) covering Sun through Pluto plus Earth, Mean/True Node,
and Mean Apogee are bundled as `data/leb2/base_core.leb2`. With `base` tier
set, calculations work offline immediately. With the default `medium` tier,
LEB2 files are auto-downloaded on first use (~119 MB).

## Horizons Backend And Calculation Modes

### Calculation modes

Four modes are available, set via `set_calc_mode()` or the `LIBEPHEMERIS_MODE`
environment variable:

| Mode | Flow | Requires |
|------|------|----------|
| `"auto"` (default) | LEB → Horizons (if no local DE440) → Skyfield | nothing (always has fallback) |
| `"skyfield"` | Always Skyfield/DE440 | DE440 downloaded |
| `"leb"` | Require LEB (auto-discovered or auto-downloaded if needed); unsupported bodies/flags fall back to Skyfield | no LEB resolvable |
| `"horizons"` | Prefer Horizons; unsupported bodies/flags fall back to Skyfield | internet connection |

In `"auto"` mode, the library resolves data transparently: it tries LEB
(bundled base-tier core, auto-discovered, or auto-downloaded for the active
tier), then falls back to the Horizons API if no local DE440 is available,
and finally to Skyfield. With the default `medium` tier, LEB2 files are
auto-downloaded on first use.

### Horizons API backend

The `HorizonsClient` provides zero-install ephemeris computation via the NASA
JPL Horizons REST API:

- LRU cache with 4,096 entries for repeated queries
- ThreadPoolExecutor with 8 workers for parallel batch fetches
- Retry with exponential backoff, 30s timeout
- Full geocentric apparent pipeline: light-time iteration, gravitational
  deflection (Sun + Jupiter + Saturn), stellar aberration, frame rotation
- Analytical dispatch for Mean Node, Mean Apogee, and Uranians (no HTTP needed)
- Precision: <0.0003" geocentric vs Skyfield reference

Supported bodies: Sun-Pluto, Earth, Chiron, Ceres, Pallas, Juno, Vesta.
Bodies not supported via Horizons (True Node, Osculating/Interpolated Apogee,
fixed stars, SEFLG_TOPOCTR) fall back to Skyfield automatically.

## Performance Milestones

### `reset_session()` -- 1750x faster consecutive calculations (a13)

Lightweight state reset that preserves file handles, LRU caches, LEB reader,
and Skyfield timescale across consecutive calculations. Only resets
per-calculation state (topocentric, sidereal mode, angles cache). Consecutive
chart calculations drop from ~3500ms to ~2ms.

### LEB fast path -- 3.1x speedup (a12)

Eliminated all Skyfield calls from the LEB calculation pipeline. The
precession-nutation matrix is built entirely from LEB-native nutation
Chebyshev data and pure-Python IAU 2006 Fukushima-Williams precession
polynomials. Single `calc_ut()` call: ~500us to ~161us. Full 14-body
chart: ~7ms to ~2.25ms.

### LEB2 v2 chunked format -- 33x faster cold start (a13)

New LEB2 format version splitting each body's Chebyshev coefficients into
10-year temporal chunks compressed independently. On first access, only the
chunk containing the requested Julian Day is decompressed. Cold-start
decompression: 1568ms (v1) to 47ms (v2).

### `set_ephe_path()` idempotent (a13)

When called with an unchanged path, `set_ephe_path()` is now a no-op -- no
file handles closed, no caches cleared. Eliminates redundant teardown when the
same path is set repeatedly (common in Kerykeion's `ephemeris_context()`).

### Vectorized search -- 5-100x speedup (a5)

`heliacal_ut()` and `lun_occult_when_glob()` inner loops replaced per-step
scalar Skyfield calls with batched numpy-vectorized computation:

| Function | Before | After | Speedup |
|----------|--------|-------|---------|
| `heliacal_ut()` Venus | 5-30s | 1-2.5s | 5-15x |
| `heliacal_ut()` Mercury | 5-30s | ~1s | 5-30x |
| `lun_occult_when_glob()` star | 10-60s | ~0.6s | 15-100x |
| `lun_occult_when_glob()` planet | 10-60s | ~0.9s | 10-65x |

### LEB base runtime -- 14x faster than Skyfield (0.22.0)

The LEB precomputed Chebyshev polynomial evaluation provides ~14x throughput
improvement over the Skyfield/JPL pipeline for standard planetary calculations.

## Known Limitations And Tradeoffs

### Inherent engine divergences

LibEphemeris uses NASA JPL DE440/DE441 via Skyfield while pyswisseph uses the
Swiss Ephemeris's internal planetary theory. These are different computational
engines that produce slightly different results:

- **Moon**: up to 3.32" divergence (different lunar theory).
- **Planets**: 0.04-0.75" (different ephemeris data and light-time iteration).
- **Hypothetical bodies**: ~35" (Keplerian propagation vs SE internal method).
- **Retrograde station timing**: up to ~83 minutes for slow outer planets
  (numerical vs analytical velocity derivatives, amplified near zero velocity).
- **Heliocentric**: ~0.01-0.03" systematic offset (SSB-Sun reference point).

### Data coverage

- **Asteroid SPK files**: safe range 1920-2080 CE. Outside this range, SPK
  data from JPL Horizons is unavailable and calculations use Keplerian
  orbital elements as fallback (substantially lower precision).
- **Extended tier extreme dates**: Meeus nutation polynomial degradation adds
  ~0.003-0.005" beyond +/-2000 years from J2000. This is a physical limit of
  the nutation series.

### Behavioral differences

- `SEFLG_MOSEPH` is accepted but silently ignored. All calculations use JPL
  DE440/DE441.
- Gauquelin sectors (G) returns 36 sectors, not 12 cusps.
- Fixed stars and `SEFLG_TOPOCTR` are not supported via the Horizons backend;
  they fall back to Skyfield automatically.
- The Horizons backend requires internet connectivity.
- LEB mode requires a configured `.leb` or `.leb2` file; raises `RuntimeError`
  if none is available and mode is set to `"leb"`.

## Getting Started

### Install

```bash
pip install libephemeris
```

This installs the library with the LEB2 base-tier core (14 bodies, ~10.6 MB)
bundled. With the default `medium` tier, additional LEB2 files are
auto-downloaded on first use (~119 MB).

### Quick start

```python
import libephemeris as eph

# Planet position
jd = eph.swe_julday(2024, 6, 15, 12.0)
result, retflag = eph.swe_calc_ut(jd, eph.SE_SUN, eph.SEFLG_SPEED)
lon, lat, dist, lon_speed, lat_speed, dist_speed = result
print(f"Sun longitude: {lon:.6f} deg")

# House cusps
cusps, ascmc = eph.swe_houses(jd, 41.9028, 12.4964, b"P")
for i, cusp in enumerate(cusps, 1):
    print(f"House {i}: {cusp:.4f} deg")

# Fixed star
result, star_name, retflag = eph.swe_fixstar2_ut("Regulus", jd, eph.SEFLG_SPEED)
print(f"Regulus: {result[0]:.6f} deg")
```

### Download additional data (optional)

```bash
# Full offline coverage (DE440 + planet centers + minor-body SPKs)
libephemeris download medium          # 1550-2650, ~200 MB total

# Faster calculations via full LEB
libephemeris download leb2-base       # LEB2 compressed (~33 MB, 1850-2150)

# Check what you have
libephemeris status
```

### Configure calculation mode

```python
import libephemeris as eph

# Use specific backend
eph.set_calc_mode("horizons")     # NASA Horizons API (zero-install)
eph.set_calc_mode("leb")          # LEB precomputed (fastest)
eph.set_calc_mode("skyfield")     # Skyfield/DE440 (reference)
eph.set_calc_mode("auto")         # Automatic fallback (default)
```

Or via environment variable: `LIBEPHEMERIS_MODE=horizons`

import libephemeris as eph
