# v1.0.0a12 Release Notes

## Performance: 3x Faster LEB Calculations

The LEB fast path now runs entirely without Skyfield. The precession-
nutation matrix — previously the single biggest bottleneck at 10-40%
of each `calc_ut()` call — is now computed from LEB-native nutation
Chebyshev data and pure-Python IAU 2006 precession polynomials.

### Benchmark: 14-body astrological chart

```
Before (a11):  ~7.0 ms/chart,  ~500 µs/call
After  (a12):  ~2.3 ms/chart,  ~161 µs/call
Speedup:       3.1x
```

### What changed

The old path created a Skyfield `Time` object on every frame-data
cache miss (~200 µs). The new path:

1. Reads dpsi/deps from LEB nutation Chebyshev (~1.5 µs)
2. Evaluates IAU 2006 Fukushima-Williams precession polynomials (~1 µs)
3. Builds the BPN rotation matrix with pure-Python `_fw2m()` (~2 µs)

Total: ~5 µs vs ~200 µs — 40x faster for this single operation.

### Precision

Identical to previous releases. Max longitude difference vs Skyfield
reference: 0.000693" (well within the <0.001" target).

### Compatibility

- `swe_calc_ut()` API: unchanged
- pyswisseph 1:1 compatibility: preserved
- LEB file format: unchanged (nutation data was already stored, just unused)
- Skyfield fallback: automatic for LEB files without nutation section
