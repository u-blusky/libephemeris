# Release Notes — LibEphemeris v0.22.0

**Release Date:** 2026-03-02
**Previous Version:** 0.20.0 (PyPI)
**Python:** >= 3.9

---

## Highlights

This is a major feature release introducing the **Binary Ephemeris Mode (LEB)**
system, **REBOUND/ASSIST N-body integration**, and significant **Keplerian
precision improvements**. LEB provides ~14x runtime speedup over the
Skyfield/JPL pipeline via precomputed Chebyshev polynomial approximations stored
in `.leb` files.

---

## New Features

### Binary Ephemeris Mode (LEB) — Complete Implementation

Full implementation of the LibEphemeris Binary (LEB) precomputed ephemeris
system. LEB files store Chebyshev polynomial approximations for 31 celestial
bodies, providing ~14x speedup with configurable precision.

#### Per-body date ranges for asteroids

Asteroid SPK kernels from JPL Horizons cover only ~1600-2500 CE, while
planetary ephemerides span much wider ranges. LEB now stores per-body
`jd_start`/`jd_end` in each `BodyEntry` header, allowing asteroids to have
narrower coverage than planets. Out-of-range queries transparently fall back
to Skyfield.

#### Vectorized analytical body generation (~10x speedup)

The 6 ecliptic bodies (True Node, True Lilith, Osculating Apogee, Interpolated
Apogee, Interpolated Perigee, Mean Apogee) previously made ~328,000 scalar
Skyfield calls each. A single vectorized Skyfield call now computes all shared
Moon ephemeris data. Analytical group generation went from ~5-8 minutes to ~38
seconds for base tier.

New vectorized functions:
- `_calc_mean_lilith_batch()` — vectorized mean apse analytical calculation
- `_calc_lunar_fundamental_arguments_batch()` — vectorized Delaunay arguments
- `_calc_elp2000_apogee_perturbations_batch()` — vectorized 40+ term series
- `_calc_elp2000_perigee_perturbations_batch()` — vectorized 61-term series
- `_eval_ecliptic_bodies_batch()` — single Skyfield call, all 6 bodies
- `generate_ecliptic_bodies_vectorized()` — batch eval + Chebyshev fitting

#### Group generation and merge workflow

New `--group {planets,asteroids,analytical}` and `--merge FILE [FILE ...]`
CLI options. Partial group files are generation-time artifacts; at runtime only
the single merged file is used. This avoids macOS `ProcessPoolExecutor`
deadlocks with C extensions.

New poe tasks for all three tiers:
```
poe leb:generate:base:groups
poe leb:generate:medium:groups
poe leb:generate:extended:groups
```

#### Calculation mode control

New `LIBEPHEMERIS_MODE` environment variable and programmatic API:

```python
from libephemeris import set_calc_mode, get_calc_mode

set_calc_mode("auto")      # Use LEB if configured, else Skyfield (default)
set_calc_mode("skyfield")  # Always use Skyfield
set_calc_mode("leb")       # Require LEB; raises RuntimeError if unavailable
```

#### LEB binary ephemeris download commands

```bash
libephemeris download:leb:base       # ~53 MB, 1850-2150 CE
libephemeris download:leb:medium     # ~175 MB, 1550-2650 CE
```

Python API:
```python
from libephemeris import download_leb_for_tier
download_leb_for_tier("medium")  # downloads + activates
```

Features: atomic downloads with SHA256 verification, progress bar, `--force`
flag, post-download LEB validation, optional auto-activation.

#### LEB auto-discovery

`get_leb_reader()` automatically discovers downloaded LEB files without
requiring explicit configuration. Resolution order:
1. Explicit path via `set_leb_file()`
2. `LIBEPHEMERIS_LEB` environment variable
3. Auto-discovery: `~/.libephemeris/leb/ephemeris_{tier}.leb`

#### Data source logging

`logger.debug()` calls at all calculation dispatch points. Activate with
`set_log_level('DEBUG')` or `LIBEPHEMERIS_LOG_LEVEL=DEBUG`.

Sources logged: `LEB`, `LEB->fallback`, `Skyfield`, `SPK`,
`SPK (auto-downloaded)`, `ASSIST (n-body)`, `Keplerian (fallback)`.

### Keplerian Precision Improvements

#### Laplace-Lagrange secular perturbations

Implemented the (h,k)/(p,q) vector formalism for secular evolution of
eccentricity and inclination under gravitational perturbations from Jupiter,
Saturn, Uranus, and Neptune. Uses Laplace coefficients for coupling terms.

#### Multi-epoch orbital elements

Generated Keplerian elements from SPK Type 21 state vectors at 50-year
intervals from 1650-2450 CE for 6 bodies. `_get_closest_epoch_elements()`
selects the element set with the smallest time offset:

| Offset    | Before   | After  | Improvement |
|-----------|----------|--------|-------------|
| 25 years  | 3.3 deg  | 2.6'   | ~75x        |
| 50 years  | 5.3 deg  | 3.6 deg| ~1.5x       |
| 100 years | 10.7 deg | 3.5 deg| ~3x         |

### REBOUND/ASSIST N-body Integration

Complete integration of REBOUND/ASSIST as a fallback for ephemeris-quality
asteroid orbit propagation:

- `propagate_orbit_assist()` — full integration with Sun, Moon, 8 planets,
  16 massive asteroids, J2/J3/J4 harmonics, and GR corrections
- Fallback chain: SPK > auto-download > ASSIST > Keplerian
- Cached ASSIST availability check with `check_assist_data_available()`

Download command:
```bash
libephemeris download:assist              # Both files (~714 MB)
libephemeris download:assist --no-asteroids  # Planet ephemeris only (~98 MB)
```

## Changes

- Removed `ProcessPoolExecutor` from LEB generator (macOS deadlock fix)
- Vectorized Skyfield ICRS evaluations (~150x speedup for individual body calls)
- Vectorized nutation computation via `erfa.nut06a()` (~50x speedup)
- `apply_secular_perturbations()` signature changed from 4-tuple to 6-tuple
- `close()` now resets `_CALC_MODE` and calls `reset_assist_data_cache()`
- Removed Swiss Ephemeris naming/references from generator labels
