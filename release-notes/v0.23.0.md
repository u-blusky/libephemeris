# Release Notes — LibEphemeris v0.23.0

**Release Date:** 2026-03-09
**Previous Version:** 0.22.0
**Python:** >= 3.9

---

## Highlights

**LEB Precision V3: sub-milliarcsecond accuracy for all 31 celestial bodies
across all three precision tiers.** This release completely rewrites the LEB
runtime pipeline, replacing the previous ICRS-to-ecliptic coordinate conversion
approach (which suffered from 1-5 arcsecond errors due to retrograde cusps and
COB oscillations) with a physics-correct pipeline that stores smooth ICRS
barycentric coordinates and applies gravitational deflection,
special-relativistic aberration, and precession-nutation at evaluation time.

**Previous worst-case error:** 4.85 arcseconds (Saturn, base tier).
**New worst-case error:** 0.000332 arcseconds (Moon, base tier) — a
**14,600x improvement**.

## New Features

### COORD_ICRS_BARY_SYSTEM (coordinate type 4)

Outer planets (Jupiter, Saturn, Uranus, Neptune, Pluto) now store their
**system barycenter** positions in ICRS coordinates. System barycenters are
ultra-smooth trajectories free of high-frequency moon oscillations that made
Chebyshev fitting unreliable for planet centers. The Center-of-Body (COB)
correction is applied at runtime using SPK data or analytical moon theories.

### PPN gravitational deflection in LEB runtime

The LEB pipeline now applies post-Newtonian (PPN) gravitational light
deflection from the Sun, Jupiter, and Saturn, matching Skyfield's `apparent()`
pipeline. This corrected a ~3.95 arcsecond systematic error on Saturn.

Formula: `delta_theta = (1+gamma) GM/(c^2 d) * (sin theta)^-1`

Three deflecting bodies:
- Sun (dominant, ~1.75" max)
- Jupiter (~0.017" max)
- Saturn (~0.006" max)

### Skyfield-compatible asteroid pipeline

New `_SpkType21Target` class in `spk.py` wraps SPK Type 21 asteroid kernels as
Skyfield `VectorFunction`-compatible objects, routing asteroids through
Skyfield's `observe()`/`apparent()` pipeline. This eliminated ~0.3-0.4"
systematic errors from the previous manual ecliptic J2000 approach.

### Precision measurement script

New `scripts/measure_precision.py` for dense end-to-end precision measurement:
- Vincenty-formula angular separation for numerically stable error measurement
- Per-body statistics: mean, P99, max error in arcseconds
- `--tier` flag with auto-detection from LEB filename
- `--group` flag for testing specific body categories
- `--samples` flag for configurable sampling density

## Pipeline Rewrite

### LEB runtime pipeline (fast_calc.py, +334 lines)

Complete rewrite of `_pipeline_icrs()` — the core evaluation pipeline. The new
"Pipeline A'" performs:

1. **Chebyshev evaluation** — Clenshaw algorithm on ICRS barycentric coefficients
2. **COB correction** (system barycenters only) — runtime offset via SPK/analytical
3. **Light-time iteration** — Newton-Raphson convergence for retarded position
4. **Gravitational deflection** — PPN formula with Sun/Jupiter/Saturn
5. **Special-relativistic aberration** — Bradley formula with Earth velocity
6. **Precession-nutation** — IAU 2006/2000A frame rotation to ecliptic of date

Previous pipeline stored geocentric ecliptic coordinates directly, which failed
at retrograde stations (cusps in longitude) and contained COB oscillations.

## Bug Fixes

- **COB evaluation time bug**: `_apply_cob_correction()` was evaluating at
  retarded time instead of observer time (one-line fix, eliminated outer planet
  residual errors)
- **Asteroid pipeline mismatch**: 0.3-0.4" systematic error from frame tie and
  nutation model differences — fixed by routing through Skyfield's pipeline
- **measure_precision.py**: three bugs causing false error reports (LEB
  auto-discovery contamination, per-sample LEBReader creation, cross-tier SPK
  mismatch)
- **Delta T precision**: replaced `reader.delta_t()` with `swe_deltat()` for
  UT->TT conversion in `fast_calc_ut()`
- **Ecliptic body generation**: was hardcoding `8d/13` for all bodies, now
  respects per-body BODY_PARAMS

## Upgrading

```bash
pip install --upgrade libephemeris
```

If using LEB files, regenerate them with the V3 pipeline:
```bash
poe leb:generate:medium:groups
```

Or download pre-built:
```bash
libephemeris download:leb:medium
```

No breaking API changes.
