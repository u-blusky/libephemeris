# LibEphemeris v1.0.0-alpha.2 — LEB2 Compressed Ephemeris Format

**Release date:** 2026-03-26

---

## Highlights

This release introduces **LEB2**, a compressed binary ephemeris format that reduces file
sizes by 5-15x while maintaining sub-arcsecond precision. The core body set (14 bodies
covering all planets, lunar nodes, and Mean Apogee) compresses to **7.7 MB** — small
enough to ship directly inside a PyPI wheel. This is the foundation for making
`pip install libephemeris` work out of the box with no additional downloads.

---

## LEB2: How It Works

### The Problem

LEB1 files store Chebyshev polynomial coefficients as raw float64 values. The base tier
(1850-2150, 31 bodies) weighs 101.8 MB — too large for PyPI's 100 MB limit. Standard
lossless compression (zstd, gzip) achieves only ~1.2x on this data because IEEE 754
float64 mantissa bits appear random to general-purpose compressors.

### The Solution: Error-Bounded Lossy Compression

LEB2 exploits a key insight: **most Chebyshev coefficients carry far more precision than
needed**. For a degree-13 polynomial fit of the Moon's trajectory:

| Coefficient | Magnitude | Bits needed for 0.001" | Bits in float64 |
|---|---|---|---|
| c0 (mean position) | ~1.0 AU | 28 | 52 |
| c1 (linear rate) | ~0.01 AU | 23 | 52 |
| c5 (5th order) | ~1e-8 AU | 4 | 52 |
| c6-c13 (high order) | < 5e-9 AU | **0** (noise floor) | 52 |

57% of the Moon's coefficients are below the precision target and can be zeroed entirely.
The remaining coefficients only need a fraction of their 52 mantissa bits.

### Compression Pipeline

```
Raw coefficients (float64)
    |
    v
[1] Mantissa truncation — zero out unneeded mantissa bits per coefficient order
    |
    v
[2] Coefficient-major reorder — group same-order coefficients across all segments
    |                           (all c0 together, all c1 together, ...)
    v
[3] Byte shuffle — transpose byte lanes (HDF5/Blosc-style) so exponent bytes
    |               and zeroed mantissa bytes cluster together
    v
[4] zstd level 19 — the regularized data now compresses excellently
    |
    v
Compressed blob (~6-15x smaller)
```

## Modular File Architecture

LEB2 introduces a modular packaging strategy. Instead of a single monolithic file
containing all 31 bodies, bodies are grouped into 4 files:

```
data/leb2/
  base_core.leb         7.7 MB   (14 bodies — PyPI wheel)
  base_asteroids.leb    7.66 MB   (5 bodies — download)
  base_apogee.leb      10.31 MB   (3 bodies — download)
  base_uranians.leb      2.02 MB   (9 bodies — download)
```

The `CompositeLEBReader` auto-discovers companion files:

```python
import libephemeris as swe

# Set path to any group file — companions are auto-discovered
swe.set_leb_file("data/leb2/base_core.leb")
swe.set_calc_mode("leb")

# Bodies from different files work transparently
swe.swe_calc_ut(2451545.0, swe.SE_SUN, swe.SEFLG_SPEED)      # from core
swe.swe_calc_ut(2451545.0, swe.SE_CHIRON, swe.SEFLG_SPEED)    # from asteroids
swe.swe_calc_ut(2451545.0, 40, swe.SEFLG_SPEED | swe.SEFLG_HELCTR)  # from uranians
```

## LEB Parameter Optimization

Independent of the LEB2 format, Chebyshev fitting parameters were optimized for
two body categories:

- **Uranians (40-48):** interval 32d/degree 13 → 256d/degree 7. These are pure
  analytical functions (Keplerian propagation) with ~0" fitting error. Saves 9.5 MB.
- **Pluto:** interval 32d/degree 13 → 64d/degree 11. 0.0005" fitting error. Saves 0.6 MB.

All other bodies were tested and documented in `proposals/leb-optimization-findings.md`.
Notable findings: Mercury (e=0.206) and asteroids cannot tolerate wider intervals due
to orbital eccentricity; Jupiter/Saturn have mutual perturbations requiring high degree.

## Poe Task Reference

```bash

poe leb2:convert:base                # All groups
poe leb2:convert:base:core           # Core only (~6.5 MB)

poe leb2:convert:medium              # Medium tier (all groups)
```
