# Release Notes — LibEphemeris v0.24.0

**Release Date:** 2026-03-16
**Previous Version:** 0.23.0
**Python:** >= 3.9

---

## Highlights

**Precision V3: sub-arcsecond accuracy across the full API surface,
comprehensive pyswisseph compatibility audit, and 30+ critical bug fixes.**

This release represents the most thorough correctness audit in the project's
history, with 42 commits covering every major subsystem: eclipses, fixed stars,
house systems, crossing functions, sidereal/ayanamsha, hypothetical bodies,
rise/set, heliacal events, and time functions.

Previous worst-case errors ranged from 2 degrees (hypothetical bodies) to 18.9"
(nutation-dependent modes). After this release, all primary computation modes
achieve sub-arcsecond agreement with the reference API, with most achieving
sub-milliarcsecond precision.

## New Features

### New calculation flags

- **SEFLG_NOGDEFL** — Skip gravitational light deflection while retaining
  aberration. Useful for comparing geometric vs apparent positions.
- **SEFLG_ICRS** — Frame bias correction (GCRS to ICRS) per IAU 2006
  Resolution B2.
- **SEFLG_SPEED3** — Numerical three-point speed computation (alternative to
  analytical derivatives).

### Eclipse Saros series

Eclipse attributes now include Saros series number (`attr[9]`) and member
number (`attr[10]`), enabling eclipse cycle analysis.

### Missing sidereal modes

Four sidereal modes that were missing from the API:
- `SE_SIDM_LAHIRI_1940`
- `SE_SIDM_LAHIRI_VP285`
- `SE_SIDM_KRISHNAMURTI_VP291`
- `SE_SIDM_LAHIRI_ICRC`

### Fixed star flag support

All SEFLG flags now work correctly with fixed star functions via shared
`_apply_fixstar_flags()` helper:

- SEFLG_SIDEREAL — ayanamsha subtraction for sidereal coordinates
- SEFLG_J2000 — J2000 ecliptic via precession back-transform
- SEFLG_NONUT — mean obliquity for equatorial transform
- SEFLG_XYZ — spherical-to-Cartesian conversion
- SEFLG_RADIANS — degree-to-radian conversion
- SEFLG_TRUEPOS — geometric position (no light-time)

### Complete swe_ time function aliases

All time functions now have `swe_` prefixed aliases for full pyswisseph API
compatibility: `swe_date_conversion`, `swe_day_of_week`, `swe_utc_to_jd`,
`swe_jdet_to_utc`, `swe_jdut1_to_utc`, `swe_utc_time_zone`, `swe_time_equ`,
`swe_lat_to_lmt`, `swe_lmt_to_lat`, `swe_sidtime`, `swe_sidtime0`.

- `swe_date_conversion` returns proper pyswisseph type:
  `(valid, jd, (year, month, day, hour))`
- Accepts bytes calendar parameter (`b'g'`/`b'j'`)
- Leap second validation: rejects `second=60` on non-leap-second dates

### EPUB manual generator

Pandoc-free EPUB generation using ebooklib+markdown with proper chapter
splitting, NCX/NAV navigation, and CSS optimized for Kobo e-readers.

New poe tasks: `docs:manual:generate`, `docs:manual:generate:it`,
`docs:manual:generate:en`.

### Documentation

- Complete English translation of the user manual (16 chapters, ~140 pages)
- Manual build pipeline for EPUB and PDF generation
- NOTICE.md: formal declaration of independent provenance and IP status
- Independent verification results against JPL Horizons and astropy/ERFA
- 233 comparison scripts reorganized into 17 semantic categories

## Bug Fixes

### Hypothetical bodies — full Keplerian propagation

Replaced simplified linear mean longitude propagation with full Keplerian
orbital mechanics for all Uranian/hypothetical planets (Cupido through
Poseidon, Transpluto):

- Newton-Raphson Kepler equation solver
- J1900-to-J2000 equinox precession using IAU 2006 model
- Gaussian gravitational constant for mean motion (`n = k/a^1.5`)
- Heliocentric-to-geocentric conversion via Skyfield
- SEFLG_HELCTR and SEFLG_J2000 flag support

**Precision: ~2 deg -> ~35" (200x improvement)**

### Fixed stars — J2000 frame and catalog

- J2000 frame uses native Skyfield `ecliptic_J2000_frame` instead of manual
  precession back-rotation, fixing ~5.4" systematic offset
- Updated proper motions for 99/116 stars from van Leeuwen 2007 new Hipparcos
  reduction (CDS/VizieR I/311/hip2)
- Fixed Algedi (Alpha-2 Cap): coordinates were for Alpha-1 Cap component
- Fixed Asellus Borealis: wrong HIP number (43103=Iota Cnc -> 42806=Gamma Cnc)
- NONUT flag subtracts dpsi for mean ecliptic output (~14" effect)
- SIDEREAL+EQUATORIAL combination fixed

### Eclipse calculations

**Solar eclipses:**
- Hybrid eclipse classification with proper threshold and re-classification
  at refined maximum
- NONCENTRAL flag handling
- Obscuration returns `(r_moon/r_sun)^2` for total eclipses (was capped at 1.0)
- Shadow width sign: negative for total (umbra), positive for annular (antumbra)
- Sunrise/sunset eclipse visibility with refraction-corrected horizon

**Lunar eclipses:**
- Shadow axis distance includes both longitude and latitude components (was
  latitude-only, causing magnitude errors up to 2.2)
- Refraction-corrected horizon threshold (-0.36 deg geometric altitude)
  for moonrise/moonset, fixing ~120s timing errors
- Apparent altitude with Skyfield atmospheric refraction model
- Removed incorrect penumbral magnitude caps (values > 1.0 are physically
  correct for full penumbral immersion)

**Other:**
- Heliocentric pheno computation returns phase angle/elongation from Earth's
  perspective (was returning zeros)
- Eclipse shadow width always computed regardless of central eclipse status

### House systems

- Vertex equator fallback for calculations near the equator
- Sidereal ascmc ayanamsha correction
- Morinus house_pos quadrant determination
- Sunshine houses_armc calculation
- Koch house_pos floating-point boundary check for bodies on MC/IC axis

### Crossing functions — sub-milliarcsecond convergence

All Newton-Raphson convergence tolerances unified to 0.001":
- Generic planet tolerance: 0.1" -> 0.001" (100x tighter)
- Moon tolerance: 0.05" -> 0.001" (50x tighter)
- Moon max iterations: 30 -> 50
- Generic planet max iterations: 50 -> 60
- Heliocentric max iterations: 60 -> 80

Timing precision improvements:
- Moon node latitude residual: 0.045" -> 0.001" (45x)
- Moon TT crossing median: 9ms -> 1.5ms (6x)
- Moon node TT crossing max: 970ms -> 13ms (75x)

Additional fixes:
- Brent fallback for retrograde planets (Mars 180 deg was failing)
- Scaled bracket search window for slow planets (Saturn/Jupiter helio)
- Full-orbit crossing for slow planets (Jupiter 0 deg, 10+ year searches)

### Rise/set and transit

- Circumpolar detection margin for fast-moving bodies (Moon at 65 deg latitude
  was falsely flagged as circumpolar due to ~13 deg/day declination change)
- Adaptive search steps for grazing conditions (10-minute steps near-grazing,
  20-minute for moderate)
- Twilight disc correction: now uses geometric center at depression angle
  (was using upper limb, causing 68-163s offsets)
- Bennett (1982) altitude-dependent refraction replaces flat 0.5667 deg

### Node/apsides and lunar theory

- Aphelion computation returns orbit second focal point (matching standard
  ephemeris behavior)
- Moon branch uses mean lunar theory values instead of osculating elements
  (fixing 1-26 deg errors)
- Planet radii updated to NASA volumetric mean values for gas/ice giants
- Moon magnitude uses piecewise Allen/Samaha photometric model (correctly
  diverges to positive infinity at new Moon)

### Sidereal time — IAU 2006 GMST

Replaced IAU 1982 polynomial (Meeus Ch.12 eq.12.4) with IAU 2006 GMST via
`erfa.gmst06()` (Capitaine et al. 2003). The old formula diverged from the
current IAU standard by up to 0.13s at 50 years from J2000.

### Ayanamsha corrections

Reference offsets corrected:
- GALEQU_TRUE: 240.0 -> 239.94708 (Hipparcos-era galactic frame)
- GALEQU_MULA: 246.62 -> 246.6137 (~32" correction)
- TRUE_SHEORAN: 178.607 -> 178.60170 (~19" correction)
- VALENS_MOON: 181.0458 -> 181.04054 (~19" correction)

Sidereal ayanamsha correction now applied to hypothetical/Uranian bodies and
Transpluto (was missing entirely).

### Heliacal API

- HELFLAG_VISLIM_DARK (was 2048, now 4096) and HELFLAG_VISLIM_NOMOON
  (was 4096, now 8192) — wrong bit positions due to missing SEARCH_1_PERIOD
- 6 missing HELFLAG constants added
- vis_limit_mag returns 10 elements (was 8)
- Proper heliacal_pheno_ut wrapper with reference-compatible signature

### Date handling

`swe_julday()` and `swe_revjul()` now use `math.floor()` instead of `int()`
for the Meeus INT() function. For negative years, `int(-30/4)=-7` but
`floor(-30/4)=-8`, causing +1 day errors for BCE dates.

`swe_revjul()` also respects the gregflag parameter for proleptic Gregorian
output (was auto-detecting calendar by JD threshold).

### Other fixes

- SEFLG_XYZ and SEFLG_RADIANS preserved in retflag (were stripped)
- Sun phase returns 0.0 in pheno_ut (inapplicable for self-luminous body)
- LEB fast path passes user ayanamsha parameters (t0, ayan_t0)
- Markdown dependency minimum lowered to 3.7 for Python 3.9 compatibility
